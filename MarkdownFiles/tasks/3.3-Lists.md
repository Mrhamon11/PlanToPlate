# Task 3.3: Generic Lists Framework

## Goal
Establish a flexible, multi-purpose list framework capable of holding raw text strings, nested recipes, or full dishes seamlessly.

## Context Needed

## Step-by-Step Instructions
1. Model a flexible `List` schema. Fields: `id`, `title`, `user`, `list_type` (Shopping, Planning, Menu, Freeform), and a rich text or JSON block for markdown-like notes.
2. Build polymorphic-like join structures or explicit link tables (`ListRecipe`, `ListDish`, `ListTextItem`) connecting distinct operational items to a generic tracking sheet.
3. Construct interactive, mobile-optimized checklist interfaces running on HTMX.
4. Verify data integrity guarantees using target-driven view validation integration tests.
