# Task 3.1: Dishes (Meal Component Grouping)

## Goal
Enable users to combine individual recipes into coordinated aggregate structural objects representing complete courses or meal events ("Dishes").

## Context Needed
* `tasks/2.2-Recipes.md` API Log

## Step-by-Step Instructions
1. Design a `Dish` model. Fields: `id`, `name`, `user`, `recipes` (Many-to-Many relationship), `rating`, `favorite`, `times_made`, and `notes`.
2. Write HTMX view components allowing a user to search their recipe database and cleanly append multiple recipes to a single Dish view interface.
3. Expose matching REST API endpoints with appropriate multi-tenant validation filters.
4. Author test sets proving that a user cannot attach another user's private recipe to their personal custom Dish.
