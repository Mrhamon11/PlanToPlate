# Task 1.2: Authentication & Multi-Tenant Privacy

## Goal
Build the security core: Custom user models, secure sign-in/sign-out state logic, persistent session storage for browser restarts, and temporary password reset screens.

## Context Needed
* `tasks/1.1-Initialization.md` API Log

## Step-by-Step Instructions
1. Define a custom user model `User` extending Django's `AbstractUser`. Ensure passwords are automatically salted via standard Argon2 or PBKDF2 configurations.
2. Implement custom login views. If a user logs in with a flag `is_temporary_password=True`, block navigation and force render a password-reset form.
3. Configure Django session middleware to support extended persistent sessions ("Remember Me" behavior) so users stay logged in when closing mobile browsers.
4. Set up comprehensive integration tests covering successful login, failed authentication, forced password mutations, and logout lifecycles.

## Verification Checklist
* [ ] Users can securely log in and persist their active sessions.
* [ ] Temporary flags redirect cleanly to forced change-password screens.
* [ ] Unauthenticated traffic is safely rejected and routed to `/login/`.
