# Task 4.1: Automated Meal Planner Engine

## Goal
Deliver the core automated meal assignment scheduler, building out a lightweight Python constraint engine that parses rule sets, samples records, and automatically populates aggregated shopping lists.

## Context Needed
* `tasks/3.3-Lists.md` API Log

## Step-by-Step Instructions
1. Extend `Dish` and `Recipe` entities to support text tags (`#protein:chicken`, `#veggie`, `#carb`).
2. Construct a specialized `MealPlan` scheduling model matching daily calendar slots across an arbitrary, user-defined block of days.
3. Write a pure-Python scheduling algorithm that reads rule configurations (e.g., maximum frequencies of specific tags) and filters matching records from the database using a random shuffling selector.
4. Implement the automated aggregation routine: Loop over every selected dish's target recipes, group identical underlying ingredients matching their base physical unit types, and write the unified, consolidated totals directly out into an auto-generated "Shopping List".

## Verification Checklist
* [ ] Generated schedules strictly respect user-defined tag exclusion criteria.
* [ ] Multi-recipe material ingredients consolidate accurately without item row fragmentation.
* [ ] Target workflows pass edge-case test validations.
