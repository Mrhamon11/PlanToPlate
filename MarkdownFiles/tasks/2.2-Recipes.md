# Task 2.2: Recipe Management & Composition

## Goal
Construct the Recipe tracking platform, mapping out multi-unit relationships linking ingredients to cooking step sheets.

## Context Needed
* `tasks/2.1-Ingredients.md` API Log

## Step-by-Step Instructions
1. Create a `Recipe` model. Fields: `id`, `title`, `user`, `instructions` (ordered text or JSON steps), `rating` (1-5 stars), `favorite` (boolean), `times_cooked` (integer tracking), and a text `notes` field.
2. Build an intermediate link model `RecipeIngredient`. Fields: `recipe`, `ingredient`, `quantity` (float), and `unit_dimension` (string representations like grams, tablespoons, cups).
3. Implement interactive forms utilizing HTMX to dynamically add or delete ingredient rows when creating or updating a recipe.
4. Construct corresponding DRF Serializers and ViewSets exposing complete CRUD patterns. Ensure strict tenancy validation checks on all read/write paths.

## Verification Checklist
* [ ] Recipes save cleanly alongside nested ingredients.
* [ ] Access violations are properly blocked with `403 Forbidden` responses.
* [ ] Star metrics and favorite state toggles function correctly.
