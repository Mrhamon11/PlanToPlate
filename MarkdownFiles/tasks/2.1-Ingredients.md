# Task 2.1: Ingredient Database Management

## Goal
Build CRUD operations for the global and user-owned Ingredient registry, featuring metric conversion infrastructure to support automated list generation down the line.

## Context Needed
* `tasks/1.2-Authentication.md` API Log

## Step-by-Step Instructions
1. Create an `Ingredient` model. Fields: `id`, `name`, `user` (nullable for systemic defaults), `base_unit_type` (Mass, Volume, Count), and a text `notes` field.
2. Implement standard Django Admin registration for backend operations.
3. Write HTMX-driven views and REST API endpoints via DRF supporting complete CRUD operations. Ensure users can only modify or delete ingredients they created. System defaults or other users' entries must return Read-Only data.
4. Code robust unit tests verifying strict workspace boundaries: User B must never update an ingredient entry created by User A.

## Verification Checklist
* [ ] Full CRUD functionality verified via HTMX interfaces.
* [ ] Unit metrics safely categorized on save.
* [ ] Workspace boundary tests pass seamlessly.
